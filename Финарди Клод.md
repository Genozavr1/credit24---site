**ПЛЮСЫ finardi.ru**

**1. Preload шрифтов (5 штук)**

<link rel="preload" href="/static/fonts/SFUIDisplay-Regular.woff2" as="font" type="font/woff2" crossorigin>

Пять шрифтов SF UI Display/Text загружаются до CSS. Браузер начинает рендерить текст без FOUT (мелькания безшрифтового текста). У нас Inter грузится через Google Fonts — стоит добавить preload.

**→ Для credit24.ru:** Добавить <link rel="preload"> для Inter woff2.

**2. Слайдер Hero с 4 продуктами**

Каждый слайд — отдельная услуга (залог недвижимости, кредит наличными, ипотека, бизнес) с тремя параметрами (сумма, срок, ставка) через SVG-иконки + две кнопки (Узнать больше / Рассчитать стоимость). Человек сразу видит конкретные цифры по каждому продукту.

**→ Для credit24.ru:** У нас статичный Hero. Можно добавить ротацию: каждые 5 секунд меняется текст и параметры (кредит наличными → залог → ипотека → рефинансирование). Или сделать это через табы над Hero.

**3. Блок «Мы в СМИ» с 9 публикациями**

РБК, Banki.ru, RB.ru, Life.ru, КП, АиФ, ОСН Медиа, Свободная Пресса, Российская газета. Каждая — карточка с логотипом СМИ (серый → цветной при наведении), заголовком и ссылкой. Реализовано через карусель data-slider="certificates".

Техническая реализация интересная: два <img> на каждую карточку — class="mono" (серый) и class="color" (цветной, display:none). При :hover серый прячется, цветной показывается. Это создаёт эффект «оживания» логотипа.

**→ Для credit24.ru:** Этот приём с серым/цветным логотипом стоит применить к карусели банков-партнёров. Серые логотипы в покое → цветные при наведении. Визуально элегантно.

**4. Карусель сертификатов банков-партнёров (25 штук)**

Абсолют, Дельта, МТС, Открытие, Совкомбанк, Россельхоз, Тинькофф, ПСБ и др. Тоже с серым/цветным эффектом. Это **сертификаты аккредитации** — сканы реальных документов от банков. Намного сильнее простых логотипов.

**→ Для credit24.ru:** Если есть сертификаты партнёрства от банков — обязательно отсканировать и добавить в карусель. Это доказательство аккредитации, которое нельзя подделать.

**5. Фото-отзывы клиентов (40+ штук)**

Рукописные отзывы на бумаге, сфотографированные. Каждый — <img> + заголовок + текст + имя. Alt-тексты привязаны к номерам договоров: alt="Отзыв по договору № 62887". Это серьёзный сигнал подлинности — номер договора = верифицируемый факт.

**→ Для credit24.ru:** Добавить фото рукописных отзывов с номерами договоров. Это мощнее любых текстовых отзывов на сайте.

**6. Schema.org — тройная разметка**

Три JSON-LD блока: FinancialService, Organization, LocalBusiness. Причём LocalBusiness содержит AggregateRating (4.7, 1001 оценка, 588 отзывов). Это правильный подход — разные типы для разных целей в Google.

У credit24.ru есть FinancialService + FAQPage + AggregateRating. Но нет отдельного Organization и LocalBusiness. Стоит добавить.

**7. UTM-трекинг на всех формах**

Каждая форма содержит скрытые поля: utm\_source, utm\_medium, utm\_campaign, utm\_term, utm\_content, referrer. Плюс отдельный скрипт utm-tracking.js. Это позволяет понять, откуда пришёл клиент, даже если он заполнил форму через час после перехода.

**→ Для credit24.ru:** Обязательно добавить UTM-трекинг во все 5 форм. Без этого невозможно оценить эффективность рекламы.

**8. Модальное окно «Увеличьте шансы на одобрение»**

После отправки основной заявки открывается modal-chance с дополнительными вопросами: сумма, цель кредита, тип занятости (официально/неофициально/нет), просрочки, регион прописки, собственность (квартира/комната/дом с выбором региона). Это квалификация лида прямо на сайте — менеджер получает готовую анкету.

**→ Для credit24.ru:** Отличная идея — после отправки телефона показать попап «Увеличьте шансы на одобрение» с 3–5 вопросами. Это повышает качество лидов и снижает время первого звонка.

**9. Google reCAPTCHA на формах**

6 отдельных reCAPTCHA-виджетов для разных форм (callus\_modal, call\_back\_section, calc\_captcha, contacts\_captcha, vacancy\_captcha, coop\_modal). Защита от спам-ботов.

**10. Страница «Кому помогли» (/cases/)**

Отдельный раздел с реальными кейсами — не просто отзывы, а истории решения проблем. Это контент-маркетинг + социальное доказательство.

**→ Для credit24.ru:** Добавить страницу /cases/ или /istorii-uspekha/ с 5–10 реальными кейсами: ситуация → что сделали → результат.

**11. Раздел «Сотрудничество»**

Четыре подраздела: агентам недвижимости, агентам, банкам, брокерам. Это B2B-направление для привлечения партнёров и рефералов.

**12. Рейтинги в футере с конкретными цифрами**

«Рейтинг на Яндекс.Картах — 4.7 на основе 1001 оценки» и «5.0 на основе 646 оценок» — прямо в <address> тегах. Два офиса с рейтингами.

**МИНУСЫ finardi.ru**

**1. jQuery 3.4.1**

<script src="https://code.jquery.com/jquery-3.4.1.min.js">

87 КБ лишнего JS. В 2026 году jQuery не нужен — всё, что он делает, можно написать на vanilla JS. У credit24.ru jQuery нет — это наше преимущество в скорости.

**2. Внешний CSS загружается после рендера**

jquery.fancybox.min.css подключается в самом конце <body> через <link>. Это вызывает FOUC (мелькание нестилизованного контента) при открытии модалок.

**3. Inline CSS в HTML**

<style>.novitem {background:#f2f7fb; border: 10px solid #eaf2f8;...}</style>

Стили для блока «Мы в СМИ» вставлены прямо в HTML вместо внешнего файла. Это затрудняет кэширование и увеличивает размер HTML.

**4. Нет lazy loading на слайдере**

Изображения в слайдере Hero (man.png, slider\_1.png, slider\_3.png, slider\_4.png) загружаются все сразу, хотя видим только первый слайд. Потенциально 2+ МБ лишних данных при первой загрузке.

**5. Нет мобильной нижней навигации**

Только кнопка «Позвонить» внизу (mobile-btn-div). Нет навигации как у domkredita.ru или binkor.ru. На мобильном пользователь теряет доступ к калькулятору и услугам.

**6. og:image указывает на SVG**

<meta property="og:image" content="https://finardi.ru/static/img/logo.svg"/>

SVG не поддерживается в og:image большинством платформ (Telegram, VK, Facebook). Нужен PNG/JPG 1200×630. У credit24.ru og:image правильный.

**7. Нет FAQ-разметки**

Есть раздел /faq/, но на главной нет FAQPage Schema.org. У credit24.ru есть — это даёт расширенные сниппеты в Google.

**8. Сторонний трекинг-скрипт**

<script src="https://fp.filkos.su/fplocator.js?site\_id=10329"></script>

Скрипт fingerprint-локатора загружается в самом начале <body> — блокирует рендеринг. Неизвестный домен вызывает вопросы безопасности.

**9. Закомментированные социальные сети**

Facebook, Instagram, LinkedIn, Twitter, ОК, МойМир — все class="hidden". VK — единственная активная. Закомментированный код загрязняет HTML.

**10. Нет калькулятора на главной**

Калькулятор вынесен на /faq/#credit-calculator. Пользователь должен уйти со страницы, чтобы посчитать платёж. У credit24.ru калькулятор встроен в главную — это правильнее.

**11. Огромный SEO-текст внизу**

Секция hypothec — стена текста про ипотечного брокера, 6 подразделов с подзаголовками. Классический SEO-текст 2015 года. Google давно обесценивает такие тексты, а пользователи их не читают.
